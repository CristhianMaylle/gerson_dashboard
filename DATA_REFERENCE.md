## Validación de Datos

### Reglas que Debe Cumplir el Backend

1. **Todos los valores están en rango [1, 5]**

   ```
   ✓ 1, 2, 3, 4, 5 - OK
   ✓ 1.5, 2.75, 4.2 - OK (decimales)
   ✗ 0, 0.5 - ERROR
   ✗ 5.5, 6 - ERROR
   ```

2. **Cantidad de valores = cantidad de fechas**

   ```
   Fechas: ["Ene. 2024", "Feb. 2024", "Mar. 2024"]  → 3
   Valores: [4.5, 4.3, 4.7]  → 3  ✓ OK
   Valores: [4.5, 4.3]  → 2  ✗ ERROR
   ```

3. **Promedio = media de valores**

   ```
   Valores: [4.77, 4.7, 3.92, 5.0]
   Media: (4.77 + 4.7 + 3.92 + 5.0) / 4 = 4.6  ✓
   ```

4. **Códigos de pregunta coinciden con indicador**

   ```
   SOC_1 → debe tener P07, P08, P09, P10  ✓
   SOC_1 → NO puede tener P11, P12  ✗
   ```

5. **Fechas en formato "Mes. Año"**
   ```
   ✓ "Ene. 2024", "May. 2026"
   ✗ "01/01/2024", "2024-01-01", "January 2024"
   ```

## SQL Example (si usa base de datos)

### Tabla de Indicadores

```sql
CREATE TABLE indicadores (
  id VARCHAR(10) PRIMARY KEY,
  nombre VARCHAR(255) NOT NULL,
  dimension VARCHAR(20),
  dim_label VARCHAR(255),
  preguntas TEXT -- JSON array: ["P07", "P08", ...]
);
```

### Tabla de Respuestas

```sql
CREATE TABLE respuestas (
  id INT PRIMARY KEY,
  indicador_id VARCHAR(10),
  pregunta_codigo VARCHAR(5),
  geo_id VARCHAR(20),
  fecha DATE,
  valor DECIMAL(3, 2),
  FOREIGN KEY (indicador_id) REFERENCES indicadores(id)
);
```

### Query para agregar datos

```sql
SELECT
  indicador_id,
  DATE_FORMAT(fecha, '%b. %Y') as fecha,
  AVG(valor) as promedio
FROM respuestas
WHERE geo_id = '06' AND indicador_id = 'SOC_1'
GROUP BY indicador_id, DATE_FORMAT(fecha, '%Y-%m')
ORDER BY fecha;
```

## Notas de Implementación

1. **Evitar múltiples llamadas al mismo endpoint**
   - Frontend cacheará resultados en memoria
   - Usar HTTP 304 o E-Tag si posible

2. **Los gráficos necesitan al menos 2 datos temporales**
   - 1 fecha = línea plana (válido pero poco útil)
   - 2+ fechas = tendencia visible

3. **Nombres de preguntas pueden variar de original**
   - Backend puede devolver textos más largos o editados
   - Frontend las mostrará exactamente como vengan

4. **El orden de preguntas importa**
   - Debe ser el mismo orden que en el array `indicador.preguntas`
   - O el orden esperado en la base de datos

5. **Promedios deben calcularse correctamente**
   - Redondear a 2 decimales como máximo
   - Considerar pesos si hay submuestras
